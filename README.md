# Analysis-of-the-AB-test-results
Банк столкнулся с низкой активностью клиентов в своем мобильном приложении. Для увеличения вовлеченности и лояльности команда продукта разработала и внедрила новый, улучшенный интерфейс (вариант B) с упрощенной навигацией, быстрыми переводами по шаблонам и персональными финансовыми советами.
Предоставлены 3 CSV-файла:
•	users_ab.csv - информация о пользователях, участвовавших в эксперименте.
o	user_id - Уникальный идентификатор пользователя
o	group - Группа А/Б-теста (A - контроль, B - тест)
o	age - Возраст
o	city - Город проживания (Москва, Санкт-Петербург, Новосибирск, Екатеринбург, Другой)
o	device - Тип устройства (Android, iOS)
o	reg_date - Дата регистрации в приложении
•	events_ab.csv - логи событий в приложении за период эксперимента.
o	user_id - Уникальный идентификатор пользователя
o	event_dt - Дата и время события
o	event_type - Тип события:
	login - Вход в приложение
	screen_view - Просмотр экрана (раздела)
	button_click - Клик на кнопку (например, "перевод", "платеж")
	transaction - Успешно завершенная финансовая операция (перевод, платеж)
	logout - Выход из приложения
•	visits_daily.csv - ежедневная агрегированная статистика по посещаемости приложения за последние 6 месяцев (для прогнозной модели).
o	date - Дата
o	visits - Количество уникальных посещений (сессий) за день
o	holiday_flag - Признак выходного/праздничного дня (1 - выходной/праздник, 0 - рабочий день)
